1. المقدمة

مرحباً بكم في بوابة إدارة برنامج بناء القدرات

تم تصميم هذا الدليل الشامل للمستخدمين لإرشادك خلال جميع الميزات والوظائف الخاصة ببوابة إدارة برنامج بناء القدرات لمؤسسة الملك خالد. تم بناء النظام لتمكين المسؤولين من إدارة مبادرات بناء القدرات بكفاءة، بما في ذلك برامج الاحتضان ومشاريع التسريع وورش العمل والشراكات مع المنظمات.

الغرض من هذا الدليل

يعمل هذا الدليل كمرجع كامل للمسؤولين، حيث يقدم تعليمات خطوة بخطوة لجميع عمليات النظام. سواء كنت مسؤولاً جديداً أو مستخدماً ذا خبرة، فإن هذا المستند سيساعدك على التنقل في المنصة بفعالية واستخدام جميع الميزات المتاحة لدعم المنظمات والشركات غير الربحية في تقديم تغيير اجتماعي واقتصادي مستدام.

حول برنامج بناء القدرات

مهمة البرنامج

يهدف برنامج بناء القدرات إلى تمكين المنظمات والشركات غير الربحية من تقديم تغيير اجتماعي واقتصادي مستدام من خلال الدعم الشامل والتدريب والإرشاد.

نظرة عامة على النظام

توفر بوابة الإدارة القدرات الرئيسية التالية:

إدارة المنظمات

  • مراجعة طلبات الانضمام
  • إدارة المنظمات المسجلة
  • تتبع ملفات المنظمات

الاحتضان والتسريع

  • إدارة نماذج البرامج
  • تتبع المشاريع والمشاركين
  • تعيين المستشارين

إدارة ورش العمل

  • إنشاء وجدولة ورش العمل
  • إدارة الدعوات
  • تتبع الحضور والتقييمات

التقارير والتحليلات

  • إنشاء تقارير شاملة
  • تتبع فعالية البرامج
  • مراقبة المؤشرات الرئيسية

نصيحة للتصفح

في جميع أنحاء هذا الدليل، ستجد صناديق معلومات ملونة مثل هذا توفر نصائح وتحذيرات وملاحظات مهمة لتعزيز فهمك واستخدامك للنظام.

2. Getting Started

2.1 Login Process

To access the Admin Portal, you must have valid administrator credentials provided by the system administrator.

Navigate to the Admin Portal

Open your web browser and go to the Admin Portal URL (provided by your organization)

Enter Your Credentials

Enter your username and password in the login form. The portal supports both Arabic and English languages. Use the language switcher in the top navigation to change the interface language.

Click Sign In

Click the "Sign In" button to access the system. If your credentials are valid, you'll be redirected to the dashboard.

Security Best Practices

  • Never share your login credentials with anyone
  • Use a strong password with at least 8 characters
  • Log out when finished, especially on shared computers
  • Change your password regularly (recommended every 90 days)

3. User Management

Admin Role Required

Overview

The User Management module allows administrators to create, modify, and manage employee accounts and their associated positions. This is a critical function for maintaining system security and access control.

Managing Employees

Access: User Profile Menu → Employees

View All Employees

The employee listing page displays all registered staff members with their details including name, email, position, and status.

  • Use the search box to find specific employees
  • Filter by position or status
  • Sort by any column header

Add New Employee

Click the "Add New Employee" button and fill in the required information:

  • Full Name: Required Employee's complete name
  • Email Address: Required Used for login (must be unique)
  • Position: Required Select from available positions
  • Phone Number: Optional Contact number
  • Role: Required Assign user role (Admin, CB Manager, etc.)
  • Initial Password: Required Temporary password (must be changed on first login)

Edit Employee Details

Click the "Edit" button next to any employee to modify their information. You can update all fields except the email address (which serves as the unique identifier).

Deactivate/Activate Employee

Use the status toggle to temporarily disable or re-enable employee accounts without deleting them. Deactivated accounts cannot log in but their data is preserved.

Important Security Notes

  • Never use weak or easily guessable passwords
  • Ensure employees change their initial password on first login
  • Regularly review and update employee access levels
  • Deactivate accounts immediately when employees leave
  • Audit employee activities periodically

4. Join Requests Management

Overview

The Join Requests module manages organizations that wish to participate in the Capacity Building Program. This includes processing new applications and maintaining records of registered corporations.

4.1 Corporation Requests

Access: Join Requests → Corporation Requests

This section handles incoming join requests from organizations (corporations, nonprofits, NGOs) seeking to participate in the program.

Viewing Requests

Access the Requests List

Navigate to Corporation Requests to see all pending and processed applications.

Review Request Details

Click on any request to view comprehensive information including:

  • Organization Information: Name, type, sector, field of work
  • Contact Details: Primary contact person, email, phone, address
  • Legal Information: Registration number, authorization authority, establishment date
  • Program Interest: Which programs they're interested in (Incubation, Acceleration, Workshops)
  • Additional Details: Organization size, number of employees, current projects
  • Attachments: Supporting documents (registration certificate, authorization documents, etc.)

Processing Requests

Review and Validate

Carefully review all submitted information and verify:

  • Completeness of information
  • Validity of legal documents
  • Alignment with program eligibility criteria
  • Authenticity of contact information

Make a Decision

Based on your review, you can:

  • Approve: Accept the organization into the program
  • Reject: Decline the application with a reason
  • Request More Information: Ask for additional details or clarification

Complete the Process

Upon approval, the organization is automatically moved to the "Registered Corporations" list and receives a notification email with login credentials.

Evaluation Criteria

Consider these factors when evaluating requests:

  • Legal Status: Valid registration and authorization
  • Sector Alignment: Organization's work aligns with program focus areas
  • Capacity to Benefit: Organization can meaningfully participate and benefit
  • Commitment: Demonstrates genuine interest and commitment to the program
  • Resource Availability: Has basic resources to engage in program activities

Search and Filter

Use the search and filter tools to efficiently manage requests:

5. Incubation Management

Overview

The Incubation program supports early-stage projects and organizations by providing structured mentorship, resources, and guidance. This module manages all aspects of the incubation journey from program design to participant graduation.

What is Incubation?

Incubation is a structured program designed to help nascent organizations and projects develop their capabilities, refine their strategies, and build sustainable operations. Participants receive one-on-one mentorship, attend specialized workshops, and gain access to resources and networks.

5.1 Incubation Models

Access: Incubations → Incubation Models

Incubation Models are templates that define the structure, duration, and requirements of different incubation programs. Each model can be customized for specific sectors or organization types.

Creating a New Model

Define Basic Information

Provide the fundamental details of the program model:

  • Model Name (English & Arabic): Required
  • Description: Required Clear explanation of the program
  • Duration: Required Program length in months
  • Target Sector: Optional Specific sectors this model serves
  • Capacity: Required Maximum number of participants per cohort

Define Program Phases

Break down the incubation journey into phases:

  • Phase 1 - Assessment: Initial evaluation and baseline
  • Phase 2 - Planning: Develop action plans and goals
  • Phase 3 - Implementation: Execute plans with support
  • Phase 4 - Evaluation: Assess progress and outcomes
  • Phase 5 - Graduation: Final review and certification

Set Requirements

Define eligibility criteria and participant commitments:

  • Eligibility requirements
  • Time commitment expected from participants
  • Prerequisites or pre-program activities
  • Required resources from participants

Configure Support Services

Specify the support provided to participants:

  • Number of mentorship sessions
  • Workshop series included
  • Resource access (tools, templates, etc.)
  • Networking opportunities
  • Financial support (if applicable)

Managing Existing Models

Action Description Impact
View See complete model details Read-only access
Edit Modify model parameters Affects future cohorts only
Duplicate Create copy for customization Creates new model
Archive Deactivate without deleting No longer available for new cohorts
Delete Permanently remove model Only if never used

Model Design Best Practices

  • Start with shorter durations (3-6 months) for pilot programs
  • Build in regular checkpoint evaluations
  • Include both group and individual activities
  • Allow flexibility for participant-specific needs
  • Define clear success metrics from the start

6. Acceleration Management

Overview

The Acceleration program is designed for more mature organizations ready to scale their impact. It provides intensive support to help organizations grow rapidly, expand their reach, and increase their effectiveness.

Incubation vs. Acceleration

Aspect Incubation 🌱 Acceleration 🚀
Target Early-stage organizations Established organizations
Focus Building foundations Scaling impact
Duration Typically 6-12 months Typically 3-6 months
Intensity Steady, foundational High-intensity, rapid
Outcomes Operational stability Exponential growth

Acceleration Module Structure

The Acceleration module mirrors the Incubation module in structure but with different focus and methodologies. All sections previously described for Incubation apply to Acceleration:

Acceleration Models

Access: Accelerations → Acceleration Models

  • Define intensive program structures
  • Set aggressive growth targets
  • Configure rapid-deployment support

Acceleration Advertising

Access: Accelerations → Advertising

  • Recruit high-potential organizations
  • Promote cohort opportunities
  • Manage competitive selection process

Acceleration Projects

Access: Accelerations → Projects

  • Track intensive growth initiatives
  • Monitor rapid scaling activities
  • Measure exponential impact

Acceleration Consultants

Access: Accelerations → Consultants

  • Engage expert advisors
  • Provide specialized expertise
  • Connect with industry leaders

Acceleration-Specific Features

Baseline Request System

Access: Accelerations → Baseline Request

The baseline assessment is a critical component of the acceleration program, establishing a comprehensive starting point to measure growth and impact.

Initiate Baseline Request

When an organization is accepted into acceleration, initiate a baseline request to collect comprehensive data about their current state.

Review Submitted Baseline

Participants submit detailed information including:

  • Organizational Metrics: Staff size, budget, geographic reach
  • Programmatic Data: Number of beneficiaries, programs offered, outcomes achieved
  • Financial Health: Revenue streams, expenses, sustainability
  • Operational Capacity: Systems, processes, technology
  • Strategic Position: Goals, challenges, opportunities

Validate and Approve

Review the baseline submission for completeness and accuracy. Request clarification or additional information as needed.

Baseline Survey Report

Access: Accelerations → Baseline Request Report

Generate comprehensive reports analyzing baseline data across all acceleration participants.

Aggregate Analysis

  • Overall cohort characteristics
  • Average starting metrics
  • Common challenges identified
  • Resource gaps

Comparative Insights

  • Sector comparisons
  • Regional differences
  • Size-based analysis
  • Maturity level distribution

Program Design

  • Identify common needs
  • Tailor support offerings
  • Group participants strategically
  • Allocate resources effectively

Benchmark Setting

  • Establish baseline averages
  • Set growth targets
  • Define success metrics
  • Plan evaluation framework

Using Baseline Data Effectively

  • Customization: Use baseline to tailor support to each organization's specific needs
  • Goal Setting: Establish realistic but ambitious targets based on starting point
  • Resource Allocation: Direct intensive support to areas of greatest need
  • Progress Tracking: Regularly compare current state to baseline to measure growth
  • Impact Evaluation: Use baseline as reference point for final impact assessment

Acceleration Best Practices

Keys to Successful Acceleration

  1. Rigorous Selection: Only accept organizations truly ready to scale
  2. Clear Expectations: Be explicit about intensity and time commitment
  3. Intensive Support: Provide frequent touchpoints and immediate assistance
  4. Cohort Model: Leverage peer learning and networking
  5. Milestone Focus: Drive toward specific, measurable achievements
  6. Expert Access: Connect with specialized consultants and advisors
  7. Resource Linkage: Facilitate connections to funding and partnerships
  8. Continuous Evaluation: Monitor progress closely and adjust quickly

7.4 Workshop Rating & Feedback

Access: Workshops → Rating & Feedback

Collect and analyze participant feedback to continuously improve workshop quality.

Feedback Collection

Deploy Evaluation Form

After workshop completion, send evaluation forms to participants. The system can:

  • Automatically email forms post-workshop
  • Provide online evaluation link
  • Track completion rates
  • Send reminders to non-respondents

Review Responses

Evaluate participant feedback on:

  • Content Quality: Relevance, depth, usefulness
  • Facilitation: Trainer effectiveness, engagement
  • Materials: Quality of handouts and resources
  • Logistics: Venue, timing, organization
  • Overall Satisfaction: Would recommend, met expectations
  • Suggestions: Open feedback for improvement

Analyze Trends

Look for patterns across workshops:

  • Average ratings by workshop type
  • Facilitator performance trends
  • Common suggestions for improvement
  • Topics requested by participants

Take Action

Use feedback to enhance future workshops:

  • Update workshop models based on feedback
  • Provide coaching to facilitators
  • Improve logistics and materials
  • Develop new workshops based on requests

Rating Scale

Rating Score Meaning Action Required
4.5 - 5.0 Excellent Maintain quality, share best practices
3.5 - 4.4 Good Minor improvements possible
2.5 - 3.4 Satisfactory Review and enhance
1.5 - 2.4 Needs Improvement Significant changes required
1.0 - 1.4 Poor Immediate intervention needed

Quality Assurance

Maintain high workshop standards:

  • Target average rating of 4.0 or higher
  • Address workshops rated below 3.5 immediately
  • Recognize and reward high-performing facilitators
  • Retire or significantly revise workshops consistently rated poorly
  • Seek detailed feedback when ratings are low

9. System Settings

Overview

System Settings allow you to configure the foundational data that supports all program operations. These settings ensure data consistency and provide standardized options throughout the system.

⚠️ Important: Settings Impact

Changes to system settings affect all areas of the platform. Exercise caution when modifying or deleting settings as they may impact existing data and reports.

Available Settings Modules

Regions & Cities

Access: System Settings → Regions & Cities

Manage the geographic hierarchy used throughout the system for organization locations, workshop venues, and regional analysis.

  • Regions: Saudi Arabia's administrative regions
  • Governorates: Sub-regions within each region
  • Cities: Cities and towns within governorates

Classification Sectors

Access: System Settings → Classification Sectors

Define and manage the sectors used to categorize organizations by their primary focus area (e.g., Healthcare, Education, Environment, Social Services).

  • Add new sectors as needed
  • Edit sector names and descriptions
  • Deactivate unused sectors
  • Organize sub-sectors under main sectors

Authorization Authorities

Access: System Settings → Authorization Authorities

Maintain the list of governmental and regulatory bodies that authorize and oversee nonprofit organizations (e.g., Ministry of Human Resources and Social Development).

  • Official authority names
  • Contact information
  • Types of organizations they oversee

Funding Sources

Access: System Settings → Funding Sources

Track common funding sources for organizations (e.g., Government Grants, Donations, Earned Revenue, Private Foundations).

Corporate Field of Work

Access: System Settings → Field of Work

Specify the specific areas of work or services provided by organizations (e.g., Youth Development, Disability Services, Community Development).

Type of Intervention

Access: System Settings → Type of Intervention

Define types of support and intervention provided through programs (e.g., Training, Consulting, Mentorship, Financial Support).

Managing Settings

All settings modules follow a similar management pattern:

View All Items

See the complete list of items in each category, with bilingual names (Arabic and English).

Add New Item

Click "Add New" and provide:

  • Name (English): Required
  • Name (Arabic): Required
  • Description: Optional
  • Sort Order: Optional Display order in dropdowns
  • Status: Active/Inactive

Edit Existing Item

Click the "Edit" button to modify any field of an existing setting.

Deactivate vs. Delete

Deactivate: Recommended approach. Item is hidden from new selections but preserved for historical data.
Delete: Only allowed if the item has never been used. Permanent removal.

Settings Maintenance Tips

  • Consistency: Use clear, consistent naming conventions
  • Bilingual: Always provide both Arabic and English translations
  • Review: Periodically review and clean up unused items
  • Hierarchy: Maintain logical groupings and parent-child relationships
  • Documentation: Add descriptions to clarify usage when needed

Data Integrity

Rule Description System Enforcement
No Duplicates Each setting item must be unique System prevents duplicate names
Required Fields Bilingual names are mandatory Cannot save without both languages
In-Use Protection Cannot delete items in active use System prevents deletion
Historical Preservation Deactivated items remain for reports Data integrity maintained

Error Messages

Common Error Messages Explained

  • "Session Expired": You've been inactive too long. Log in again.
  • "Access Denied": Your role doesn't have permission. Contact admin.
  • "Record Not Found": Item may have been deleted. Refresh the page.
  • "Duplicate Entry": Item already exists. Use a different name.
  • "Cannot Delete": Item is in use. Deactivate instead of deleting.
